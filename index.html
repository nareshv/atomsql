Introduction

AtomSQL is Simple Database Query Language for SQL Like systems.

Supported Features

  • ALL CRUD Operations are supported

  • Few Table metadata operations are supported

  • MySQL, Sqlite✳, Postgresql✳ are supported

  • Perl, PHP✳, Nodejs✳ are supported

Why ?

Current SQL is way too much to type for a UNIX/Linux user. So, came up with this simpler syntax to do most of the common tasks.

SQL Replacement ?

SQL itself is very complex and AtomSQL is not meant to be a replacement for it. Many Projects like Oracle, Postgresql, MySQL have invested lot of hours building a flexible SQL as per standards. Replacing all that effort in couple of weeks is not the idea.

TD;LR

AtomSQL libraries provide mechanism to convert AtomSQL queries to normal SQL queries.

AtomSQL Syntax

Below sections explain the syntax with few examples.

Table/Metadata

Few CLUES to Remember

  • = is an operator
  • + is ADD operator
  • - is DROP operator
AtomSQL MySQL
hellotable DESCRIBE hellotable
= hellotable SHOW CREATE TABLE hellotable
hellotable +abc int ALTER TABLE ADD COLUMN abc int
hellotable -abc ALTER TABLE DROP COLUMN abc

Table/Create

Few CLUES to Remember

  • {} are borrowed from C/C++ Structure Definition. So, you can remember that way.
AtomSQL MySQL
hellotable { a int, b char } CREATE TABLE hellotable ( a int, b char )
hellotable { a int, b char } engine=innodb charset=utf8 CREATE TABLE hellotable ( a int, b char ) engine=innodb charset=utf8

Table/Select

Few CLUES to Remember

  • <> describe the column selection group
  • () describe the WHERE clause
  • [] describe the row selection with array slicing. LIMIT and OFFSET
AtomSQL MySQL
hellotable <a,b> SELECT a,b FROM hellotable
hellotable <a,b> (a > b) SELECT a,b FROM hellotable WHERE (a > b)
hellotable <a,b> (a > b) [100] SELECT a,b FROM hellotable WHERE (a > b) LIMIT 100
hellotable <a,b> (a > b) [100, 200] SELECT a,b FROM hellotable WHERE (a > b) LIMIT 100 OFFSET 200

Table/Update

Few CLUES to Remember

  • // is like regular-expressions for substitution. Read perl, awk for some intro
  • () describe the WHERE clause
  • [] is again array slicing. Used for LIMIT
AtomSQL MySQL
hellotable /a = 1, b = 2/ UPDATE hellotable SET a = 1, b = 2
hellotable /a = 1, b = 2/ [50] UPDATE hellotable SET a = 1, b = 2 LIMIT 50
hellotable /a = 1, b = 2/ (a > 1) UPDATE hellotable SET a = 1, b = 2 WHERE (a > 1)
hellotable /a = 1, b = 2/ (a > 1) [25] UPDATE hellotable SET a = 1, b = 2 WHERE (a > 1) LIMIT 25

Table/Insert

Few CLUES to Remember

  • = Assignment operator. COPY TABLE STRUCTURE
  • < UNIX Redirection operator. Used to copy data from RH to LH
  • <> Column Grouping operator
  • () WHERE clause
  • = () TUPLE representation
  • = [(), ()] ARRAY of TUPLE representation
AtomSQL MySQL
new_hellotable = hellotable CREATE TABLE new_hellotable LIKE hellotable
new_hellotable <*> < hellotable <*> REPLACE INTO new_hellotable (*) SELECT (*) FROM hellotable
new_hellotable <a,b> < hellotable <a,b> REPLACE INTO new_hellotable (a,b) SELECT (a,b) FROM hellotable
new_hellotable <a,b> << hellotable <a,b> INSERT INTO new_hellotable (a,b) SELECT (a,b) FROM hellotable
new_hellotable <a,b> < hellotable <a,b> (a > 9) REPLACE INTO new_hellotable (a,b) SELECT (a,b) FROM hellotable WHERE (a > 9)
new_hellotable <a,b> << hellotable <a,b> (a > 9) INSERT INTO new_hellotable (a,b) SELECT (a,b) FROM hellotable WHERE (a > 9)
hellotable = [ (1,'c') ] INSERT INTO hellotable VALUES (1,'c')
hellotable = [ (1, 'a'), (2, 'b') ] INSERT INTO hellotable VALUES (1, 'a'), (2, 'b')
hellotable = [ (1, 'a'), (2, 'b'), (3, 'c') ] INSERT INTO hellotable VALUES (1, 'a'), (2, 'b'), (3, 'c')
hellotable <a,b> = [ (1,'c') ] INSERT INTO hellotable (a,b) VALUES (1,'c')
hellotable <a,b> = [ (1, 'a'), (2, 'b') ] INSERT INTO hellotable (a,b) VALUES (1, 'a'), (2, 'b')
hellotable <a,b> = [ (1, 'a'), (2, 'b'), (3, 'c') ] INSERT INTO hellotable (a,b) VALUES (1, 'a'), (2, 'b'), (3, 'c')

Table/Delete

Few CLUES to Remember

  • ! BANG operator
  • [] Array Slicing operator. for LIMIT
  • () WHERE clause
  • !! DROP table operator
AtomSQL MySQL
! hellotable TRUNCATE TABLE hellotable
! hellotable [10] DELETE FROM hellotable LIMIT 10
! hellotable (a > 100 and b != 'C') DELETE FROM hellotable WHERE (a > 100 and b != 'C')
! hellotable (a > 100 and b != 'C') [10] DELETE FROM hellotable WHERE (a > 100 and b != 'C') LIMIT 10
!! hellotable DROP TABLE hellotable

Table/Backup

Few CLUES to Remember

  • <> Column selection operator
  • [] Array slicing operator
  • > UNIX like redirection symbol
AtomSQL MySQL
hellotable <*> > '/tmp/backup-hellotable.csv' SELECT * INTO OUTFILE FROM hellotable
hellotable <a,b> > '/tmp/backup-hellotable.csv' SELECT a,b INTO OUTFILE '/tmp/backup-hellotable.csv' FROM hellotable
hellotable <a,b> [10] > '/tmp/backup-hellotable.csv' SELECT a,b INTO OUTFILE '/tmp/backup-hellotable.csv' FROM hellotable LIMIT 10
hellotable <a,b> [10, 20] > '/tmp/backup-hellotable.csv' SELECT a,b INTO OUTFILE '/tmp/backup-hellotable.csv' FROM hellotable LIMIT 10 OFFSET 20
hellotable <a,b> (a = 100 and a != 20) [10, 20] > '/tmp/backup-hellotable.csv' SELECT a,b INTO OUTFILE '/tmp/backup-hellotable.csv' FROM hellotable LIMIT 10 OFFSET 20

Table/Restore

Few CLUES to Remember

  • < UNIX like redirection symbol
  • << Custom redirection symbol
AtomSQL MySQL
hellotable < '/tmp/backup-hellotable.csv' LOAD DATA LOCAL INFILE '/tmp/backup-hellotable.csv' REPLACE INTO TABLE hellotable
hellotable << '/tmp/backup-hellotable.csv' LOAD DATA LOCAL INFILE '/tmp/backup-hellotable.csv' REPLACE INTO TABLE hellotable
hellotable &< '/tmp/backup-hellotable.csv' LOAD DATA LOCAL INFILE '/tmp/backup-hellotable.csv' REPLACE IGNORE INTO TABLE hellotable
hellotable &<< '/tmp/backup-hellotable.csv' LOAD DATA LOCAL INFILE '/tmp/backup-hellotable.csv' REPLACE IGNORE INTO TABLE hellotable
